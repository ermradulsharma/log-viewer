## Log files pattern

```php
<?php

use Ermradulsharma\LogViewer\Contracts\Utilities\Filesystem;

return [
    // ...

    /* ------------------------------------------------------------------------------------------------
     |  Log files pattern
     | ------------------------------------------------------------------------------------------------
     */
    'pattern'       => [
        'prefix'    => Filesystem::PATTERN_PREFIX,    // 'laravel-'
        'date'      => Filesystem::PATTERN_DATE,      // '[0-9][0-9][0-9][0-9]-[0-9][0-9]-[0-9][0-9]'
        'extension' => Filesystem::PATTERN_EXTENSION, // '.log'
    ],

    // ...
];
```

## Route

```php
<?php

return [
    // ...

    /* -----------------------------------------------------------------
     |  Route settings
     | -----------------------------------------------------------------
     */

    'route'         => [
        'enabled'    => true,

        'attributes' => [
            'prefix'     => 'log-viewer',

            'middleware' => env('LOGVIEWER_MIDDLEWARE') ? explode(',', env('LOGVIEWER_MIDDLEWARE')) : null,
        ],

        'show' => 'log-viewer::logs.show',
    ],

    // ...
];
```

By default no middleware is added to `log-viewer` route. If you need middlewares you just have to add a `LOGVIEWER_MIDDLEWARE` key to your `.env` file and add middlewares as comma separated values (no space).

- **Example 1:** single middleware

```
LOGVIEWER_MIDDLEWARE=web
```

- **Example 2:** multiple middlewares

```
LOGVIEWER_MIDDLEWARE=web,auth,custom-middleware
```
